# 古风言情写作 Skill

从提词器到成稿到自检的一站式古风言情写作工具。写作完成后自动调用 `de-ai-novel-skill` 检测并修改，确保 AI 感 <10/100。

---

## 核心理念

好的古风言情不是"用古风词汇写现代故事"，而是**用古人的思维方式讲古人的故事**。

| 维度 | AI 写法 | 人味写法 |
|------|---------|----------|
| 开头 | "这是一个风和日丽的春天" | 直接写感官/动作，不要环境描写开头 |
| 比喻 | "目光温润如春日静水" | "那眼神跟后厨刘妈数银子一模一样" |
| 情感 | "她心中五味杂陈" | "她攥了攥拳头，又松开，反复了三次" |
| 对白 | "我认为此事需要慎重考虑" | "这事儿不对劲，得重新合计合计" |
| 结尾 | "新的一天开始了" | "门外的脚步声越来越近" |

## 五步写作流程

```
第一步：选题定型（选题材、定冲突、定人物原型）
    ↓
第二步：大纲设计（剧情骨架 + 伏笔网络 + 情感节拍）
    ↓
第三步：提词生成（用 SKILL.md 中的提词器生成初稿）
    ↓
第四步：自检修改（调用 de-ai-novel-skill 检测，自动修改到 AI 感 <10）
    ↓
第五步：润色定稿（人工微调、统一风格、确认节奏）
```

### 第一步：选题定型

用 `spec/plot-templates.json` 中的题材模板选择：

| 题材 | 核心冲突 | 代表作 |
|------|----------|--------|
| 重生复仇 | 前世被害，重生扭转命运 | 知否、嫡女重生 |
| 宫斗宅斗 | 深宅权力博弈 | 锦心似玉、庶女攻略 |
| 先婚后爱 | 被迫成婚到相知相爱 | 知否、星汉灿烂 |
| 江湖武侠 | 江湖恩怨中的爱恨情仇 | 琅琊榜、有匪 |
| 仙侠修真 | 修仙路上的情劫 | 花千骨、三生三世 |

### 第二步：大纲设计

**剧情骨架模板**（以重生复仇为例）：

```
1. 死亡场景（开篇钩子，3-5 句）
2. 重生醒来（感官确认，不要直接说"我重生了"）
3. 前世碎片（用感官触发，不要线性叙述）
4. 确定时间节点（用具体物件/事件锚定）
5. 第一个小行动（试探性改变，不是直接摊牌）
6. 遇到关键人物（情绪张力）
7. 布局第一步（利用信息差）
8. 意外发现（前世不知道的真相）
9. 盟友出现（有试探过程）
10. 章末悬念（未完成的事）
```

**伏笔网络设计**：

```markdown
## 伏笔清单

| 伏笔 | 前文位置 | 后文揭示 | 类型 |
|------|----------|----------|------|
| 碗沿缺口 | 第1章 | 第5章（七岁记忆） | 物件 |
| 左肩缩 | 第1章 | 第8章（前世疤痕） | 动作 |
| 月白色 | 第1章 | 第3章（端阳宴） | 颜色 |
| "记性好" | 第2章 | 第10章（她确实记得） | 对话 |
```

### 第三步：提词生成

详见 `SKILL.md`——完整的提词器模板，按章节结构分段生成。

### 第四步：自检修改

```bash
# 自动检测并修改，直到 AI 感 <10/100
node tools/selfcheck.js <小说文件.txt>

# 指定目标分数
node tools/selfcheck.js <小说文件.txt> --target=10

# 最大迭代次数
node tools/selfcheck.js <小说文件.txt> --max-iter=5

# 输出修改后的文件
node tools/selfcheck.js <小说文件.txt> --output=<输出文件.txt>

# 只检测不修改
node tools/selfcheck.js <小说文件.txt> --dry-run
```

**自动修复能力**：
- 自动修复能将 AI 感从 60+ 降到 20 左右
- 低于 20 需要人工微调（调整段落结构、章末悬念、删减冗余形容词）
- 低于 10 需要结合提词器从源头控制（用 SKILL.md 的约束条件生成初稿）

### 第五步：润色定稿

自检通过后的人工微调清单：
- [ ] 通读全文，检查节奏是否自然
- [ ] 确认每个角色的说话方式是否一致
- [ ] 确认伏笔是否都有回收（或有意留白）
- [ ] 确认结尾是否有悬念
- [ ] 朗读测试——读出声来是否自然

## 十条铁律

写作过程中必须遵守，违反任何一条都会被自检工具检测出来：

| 编号 | 铁律 | 原因 |
|------|------|------|
| R01 | 第一句话必须是感官或动作 | AI 用环境描写开头，人类直接写感官 |
| R02 | 每章至少 2 个闲笔 | AI 每个细节都有功能，人类有"无用"细节 |
| R03 | 比喻必须来自角色经验 | AI 的比喻来自通用意象库 |
| R04 | 全文至少 1 处直接讲述 | AI 100% Show，人类 Show+Tell 混合 |
| R05 | 情感节拍不能太整齐 | AI 的情感变化像节拍器 |
| R06 | 时间标记不能太精确 | AI 喜欢"三天后"，人类更模糊 |
| R07 | 每个角色必须有独特说话方式 | AI 所有角色说话一样 |
| R08 | 禁止使用 AI 高频词 | 详见 de-ai-novel-skill 禁词表 |
| R09 | 段落长度必须有变化 | AI 段落长度均匀 |
| R10 | 结尾必须有未完成的事 | AI 喜欢总结收尾 |

## 对白规则

### 古白话定位

古风对白不是文言文，也不是现代口语，是介于两者之间的"古白话"：

| 场景 | 风格 | 示例 |
|------|------|------|
| 正式场合 | 用"您"、称呼用全名或头衔 | "臣女沈昭宁，见过侯爷。" |
| 半正式 | 用"你"、称呼用名字或排行 | "姑娘，老太太那边传话了。" |
| 私下场合 | 口语化、有打断、有方言 | "你又把柴火弄湿了？" |

### 称呼体系

| 关系 | 自称 | 称呼对方 |
|------|------|----------|
| 女子对长辈 | 臣女/民女/妾身 | 老太太/老夫人/太太 |
| 女子对同辈 | 我 | 姑娘/小姐/姐姐/妹妹 |
| 仆人对主子 | 奴婢/奴才/小的 | 姑娘/公子/老爷 |
| 男子对同辈 | 我/在下 | 兄台/公子/X郎 |

### 禁止出现的现代词

OK、好的、没问题、收到、了解、随便、加油、努力、崩溃、无语、尴尬、老公、老婆、亲爱的、拜拜、再见、不好意思、抱歉、对不起

替换：好的→好/嗯/知道了，没问题→无妨/不碍事，不好意思→失礼了，对不起→对不住

## 闲笔写作指南

闲笔 = 跟情节无关但让文本更有"人味"的描写。

| 闲笔类型 | 示例 | 作用 |
|----------|------|------|
| 环境闲笔 | "树上有个鸟窝，鸟窝里有三颗蛋" | 让空间更真实 |
| 角色闲笔 | "她喝茶前总要吹三下" | 让角色更立体 |
| 叙述者闲笔 | "这事儿说起来挺荒唐" | 让叙述者有存在感 |
| 感官闲笔 | "空气里有股潮湿的霉味" | 让读者有沉浸感 |
| 时间闲笔 | "那只猫在墙头蹲了大概一炷香的工夫" | 让时间有质感 |

**规则**：每章 2-4 个闲笔，不能太多；闲笔不能出现在高潮段落；闲笔不需要"回收"。

---

## 留白写作指南

留白 = 不写，让读者自己体会。

| 留白类型 | 示例 |
|----------|------|
| 情感留白 | "他走了。她站在门口，看着他的背影越来越小。"（不写她难不难过） |
| 对话留白 | ""我其实——"她的话卡在嗓子里，最终什么也没说。" |
| 结局留白 | "她把信折好，塞进袖子里。门外的脚步声越来越近。" |
| 细节留白 | "碗沿有一道细细的缺口。"（不解释为什么） |

## 使用工具

### 写作生成

```bash
# 用提词器生成一章初稿
node tools/write.js --genre=rebirth --chapter=1

# 指定人物原型
node tools/write.js --genre=rebirth --female=resilient --male=cold

# 指定场景
node tools/write.js --scene=waking --setting=boudoir
```

### 自检修改

```bash
# 自动检测并修改，直到 AI 感 <10/100
node tools/selfcheck.js <小说文件.txt>

# 指定目标分数
node tools/selfcheck.js <小说文件.txt> --target=10

# 输出修改后的文件
node tools/selfcheck.js <小说文件.txt> --output=<输出文件.txt>
```

### 检测 AI 味

```bash
# 单独检测（不修改）
node ../de-ai-novel-skill/tools/detect.js <小说文件.txt>

# 查看改写建议
node ../de-ai-novel-skill/tools/humanize.js <小说文件.txt>
```
