# AI 生成小说「去 AI 化」

将 AI 生成的小说文本洗掉机器话术、逻辑断层、模板感、流水账，使其读起来像真人写的。

基于知乎 AI 高频词汇表、Reddit r/ChatGPTPro 写作社区、Sudowrite/NovelAI 等工具的最佳实践，以及中文网文/文学写作标准总结而成。

---

## 核心理念

AI 写小说有五大通病：

| 通病 | 表现 | 解法 |
|------|------|------|
| **机器话术** | 大量使用 AI 高频词（"不禁"、"五味杂陈"、"目光深邃"） | 禁词表 + 替换方案 |
| **逻辑断层** | 场景切换生硬，因果链不完整，角色动机突兀 | 逻辑链检查 + 衔接补写 |
| **模板感** | 段落结构雷同，节奏均匀，每段都是"起承转合" | 打破对称 + 变奏节奏 |
| **流水账** | 平铺直叙无重点，事件罗列无张力 | 详略控制 + 场景化改写 |
| **角色脸谱化** | 所有人说话方式一样，情感表达靠固定词组 | 角色语言档案 + 差异化对白 |

## 去 AI 化工作流

### 五步流水线

```
第一步：大纲规划（AI 当思考伙伴，不当代笔）
    ↓
第二步：初稿生成（带约束条件：禁用词、五感要求、角色语言差异）
    ↓
第三步：结构审读（检查逻辑链、节奏、角色一致性）
    ↓
第四步：去 AI 化改写（针对高频词、模板句式、流水账逐项清洗）
    ↓
第五步：润色定稿（统一风格、调节节奏、打磨细节）
```

### 每步使用的 Prompt 策略

| 步骤 | AI 角色 | 核心指令 |
|------|---------|----------|
| 大纲 | 故事架构师 | "问我在写之前，不要直接给答案" |
| 初稿 | 写手 | "遵循禁词表，每场景至少三种感官描写" |
| 审读 | 编辑 | "只指出问题，不改写，用具体行号标注" |
| 改写 | 改稿师 | "只改有问题的句子，保持原文风格" |
| 润色 | 文字匠 | "朗读测试：这句话读出声自然吗？" |

## 五大改写技法

### 技法一：五感注入法

**原则**：每个场景至少涉及三种感官（视、听、嗅、味、触）。

**改写示例**：

```
❌ AI 原文：
他走进房间，感到一阵紧张。房间里很安静，他不禁有些害怕。

✅ 改写后：
他推开门，门轴发出一声闷响。空气里有股潮湿的霉味，混着旧报纸的油墨气息。
他的手指不自觉地攥紧了门把手，指节发白。天花板上的灯管嗡嗡响着，忽明忽暗。
```

**操作规则**：
- "感到 X" → 写出身体的具体反应
- "很安静" → 写出安静中能听到什么（自己的心跳、远处的车声、地板的吱呀）
- "很冷" → 写出冷的具体感受（鼻尖发麻、呼出白气、手指僵硬）

### 技法二：对白去模板化

**原则**：每个角色必须有独特的说话方式。

**角色语言档案模板**：

```json
{
  "角色名": "张三",
  "年龄": 35,
  "职业": "出租车司机",
  "教育": "高中",
  "方言": "东北口音",
  "口头禅": "那可不",
  "说话特点": "短句多、爱用反问、偶尔蹦脏话",
  "禁用": "书面语、成语、长复句"
}
```

**改写示例**：

```
❌ AI 原文（所有人说话都一样）：
张三说："我认为这件事情的处理方式需要重新考虑。"
李四说："我同意你的看法，这确实是一个需要认真对待的问题。"

✅ 改写后（角色语言差异化）：
张三说："这事儿不对劲，得重新合计合计。"
李四点了根烟，半天才开口："你说得对，可问题是——谁去说？"
```

**对白规则**：
- 禁止角色说他们不会说的话（教授不会说"那可不"，出租车司机不会说"此言差矣"）
- 对白中加入打断、犹豫、未说完的句子
- 用动作替代"XX说"：他把烟摁灭了，"不去了。"
- 潜台词比明说更有力：角色说"没事"的时候，往往有事

### 技法三：节奏变奏法

**原则**：打破 AI 的均匀节奏，制造快慢交替。

**AI 的节奏问题**：
- 每段长度相近（150-200字）
- 每段结构相同（叙述→描写→心理）
- 节奏均匀，没有快慢变化

**变奏规则**：

| 场景类型 | 句子 | 段落 | 详略 |
|----------|------|------|------|
| 紧张/打斗 | 短句、碎句 | 短段落（1-3句） | 详写，放慢时间 |
| 日常/过渡 | 中等长度 | 中等段落 | 略写，一笔带过 |
| 回忆/沉思 | 长句、绵延 | 长段落 | 详写，沉浸感 |
| 惊讶/转折 | 突然短下来 | 单句成段 | 详写，定格瞬间 |

**改写示例**：

```
❌ AI 原文（节奏均匀）：
他走到了门口，犹豫了一下，还是推开了门。房间里很暗，他摸索着找到了灯的开关。
灯亮了，他看到了房间里的景象，不禁大吃一惊。房间里一片狼藉，到处都是散落的文件。

✅ 改写后（节奏变奏）：
他走到门口。
手搭上门把手，又放下了。
再搭上。
推开门。暗的。他摸到开关，啪嗒——
灯亮了。
他愣在原地。
文件散了一地。桌子翻了。窗帘被扯掉一半，露出后面被凿穿的墙壁。
```

### 技法四：流水账→场景化

**原则**：把"然后……然后……然后……"变成有画面、有冲突、有感官的场景。

**改写示例**：

```
❌ AI 原文（流水账）：
他起床后吃了早饭，然后去了公司。到了公司后他发现大家都在议论纷纷。
他问了同事才知道发生了什么事。原来是公司要裁员了。他感到很担心。

✅ 改写后（场景化）：
闹钟响了三遍他才爬起来。厨房里昨晚的碗还没洗，他懒得管，抓了两片干面包就出了门。
电梯里遇到隔壁的老王，老王看了他一眼，欲言又止。
到了办公室，气氛不对。平时闹哄哄的格子间安静得像考场，几个人凑在一起低声说话，看到他来了又散开。
"怎么了？"他问对面的小李。
小李往经理办公室的方向努了努嘴："裁员，名单下午出。"
他把面包放下，突然没了胃口。
```

**场景化检查清单**：
- [ ] 有没有具体的地点描写？
- [ ] 有没有人物的具体动作（不是"他做了XX"，而是怎么做的）？
- [ ] 有没有感官细节（看到/听到/闻到/触到什么）？
- [ ] 有没有冲突或张力（哪怕很小）？
- [ ] 有没有删除"然后"、"接着"、"随后"？

### 技法五：逻辑链补全

**原则**：AI 常见的逻辑断层必须补上因果链。

**常见断层类型**：

| 断层类型 | 表现 | 补法 |
|----------|------|------|
| 动机断层 | 角色突然做某事，没有铺垫 | 补上前文伏笔或内心活动 |
| 情感断层 | 情绪突变，没有过渡 | 补上情绪渐变过程 |
| 时间断层 | "三天后"突然出现，没有交代 | 补上中间发生了什么 |
| 空间断层 | 角色突然换了地方，没有移动描写 | 补上转移过程 |
| 信息断层 | 角色突然知道某事，没有信息来源 | 补上谁告诉他的/怎么发现的 |

**改写示例**：

```
❌ AI 原文（动机断层）：
李明突然决定辞职。他收拾了东西，走出了公司大门。

✅ 改写后（补全动机链）：
李明在工位上坐了十分钟，一个字也没打出来。
他想起上周五老板说的话："你这个方案不行，重做。"这是第七次了。
他打开抽屉，看到里面放了三年前入职时拍的合影。照片上的自己笑得很用力。
他把照片放回去，关上抽屉。然后打开电脑，写了一封辞职信。
```

## 检查清单

### 每章完成后的自检

#### 一、禁词检查
- [ ] 全文搜索禁词表中的词，逐个处理
- [ ] 每个情感词是否都改成了具体描写？
- [ ] 每个连接词是否都必要？能否删除？

#### 二、五感检查
- [ ] 每个场景至少涉及几种感官？
- [ ] 有没有"感到XX"这种抽象表达？改成身体反应
- [ ] 环境描写是否服务于情绪/氛围？

#### 三、对白检查
- [ ] 遮住角色名，能分辨出是谁在说话吗？
- [ ] 有没有角色说了他不会说的话？
- [ ] 对白中有没有潜台词？
- [ ] 有没有"XX说"过多？用动作替代部分

#### 四、节奏检查
- [ ] 段落长度有没有变化？
- [ ] 紧张处是否用了短句？舒缓处是否用了长句？
- [ ] 有没有删除不必要的过渡段？

#### 五、逻辑检查
- [ ] 角色的每个行为有没有动机？
- [ ] 情感变化有没有过渡？
- [ ] 信息传递有没有来源？
- [ ] 场景切换有没有衔接？

#### 六、流水账检查
- [ ] 有没有"然后……然后……然后……"结构？
- [ ] 重要场景是否展开了？（不是"他做了XX"，而是怎么做的）
- [ ] 不重要的事是否一笔带过了？（不要给无关事件同等篇幅）

### 各项具体检查方法

**1. 副词检查**
```bash
# 搜索高频副词
grep -n "微微\|淡淡\|缓缓\|轻轻\|慢慢\|悄悄\|默默\|静静" novel.txt
```
- 每页 ≤ 2 个为合格
- 超过的部分删除或改为具体描写

**2. 长句检查**
- 手动通读，遇到喘不过气的句子就拆
- 超过 40 字的句子必须拆成两句以上
- 连续三段长度相近（±20字）需要调整

**3. 情绪直白度检查**
```bash
# 搜索直白情绪词
grep -n "感到\|觉得\|心中\|心里\|不禁\|不由得\|不由自主\|暗自\|强忍" novel.txt
```
- 每千字 ≤ 1 个为合格
- 全部改为身体反应/动作/神态

**4. 对白自然度检查**
- 遮住角色名，能分辨谁在说话 → 合格
- 对白中有"……"（停顿）、"——"（打断）、动作穿插 → 合格
- 连续三句纯对白无动作描写 → 不合格

**5. 景物相关性检查**
- 每处景物描写问自己：删掉这段影响剧情吗？
- 不影响 → 删除
- 影响但太长 → 缩短到一两句

**6. 视角一致性检查**
- 确认全文是第几人称
- 确认视角人物是谁
- 中途切换必须有明确标记（章节分隔、空行）

## 快速改写公式

当不确定怎么改时，套用以下公式：

### "感到 X" → 身体反应

```
公式：[身体部位] + [具体动作] + [感官细节]

感到紧张 → 手指不自觉地攥紧了衣角
感到害怕 → 后背一阵发凉，汗毛竖了起来
感到高兴 → 嘴角怎么也压不下去
感到悲伤 → 鼻子一酸，眼眶发烫
感到愤怒 → 牙关咬得咯咯响，太阳穴突突地跳
```

### "他做了X" → 场景展开

```
公式：[动作分解] + [环境反馈] + [感官细节]

他吃了饭 → 他扒了两口饭，米粒硬邦邦的，菜凉了，油花凝成白色的膜
他走了过去 → 他踩着碎石子走过去，鞋底嘎吱嘎吱响，路灯把影子拉得老长
她哭了 → 她没出声，只是肩膀一抖一抖的，眼泪掉在手背上，温热的
```

### "然后" → 场景切换

```
公式：删除"然后"，用以下方式替代：
1. 直接切场景（空一行）
2. 用时间标记："三天后"、"入秋的时候"
3. 用空间标记："回到出租屋"、"推开酒吧的门"
4. 用感官过渡："电话响了"、"门铃响了"、"雨停了"
```
