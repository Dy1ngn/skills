# AI 生成小说「去 AI 化」

检测并修复 AI 生成小说中的机器话术、逻辑断层、模板感，使其读起来像真人写的。

> 本 skill 是**第二创作者**（检测+修复），写作指导请参见对应的写作 skill（如 `ancient-romance-skill`）。

---

## 核心理念

AI 写小说有五大通病，本工具负责检测和修复：

| 通病 | 检测方式 | 修复方式 |
|------|----------|----------|
| **机器话术** | 禁词扫描（`detect.js`） | 自动替换 + 人工建议（`humanize.js`） |
| **逻辑断层** | 因果链/时间线/空间标记检测 | 人工补写衔接 |
| **模板感** | 段落长度方差/句式单调/排比检测 | 人工打散重组 |
| **流水账** | 信息密度/详略检测 | 人工场景化改写 |
| **角色脸谱化** | 对白标签/角色语言差异检测 | 人工建立角色语言档案 |

---

## 使用工具

### 检测 AI 味

```bash
# 扫描文本中的 AI 高频词，输出 AI 味评分
node tools/detect.js <小说文件.txt>

# 只检测中文
node tools/detect.js <小说文件.txt> --lang=zh

# 输出 JSON 格式（便于程序处理）
node tools/detect.js <小说文件.txt> --format=json
```

### 获取改写建议

```bash
# 对检测到的问题给出具体改写建议
node tools/humanize.js <小说文件.txt>

# 自动替换可替换的部分，输出到新文件
node tools/humanize.js <小说文件.txt> --apply

# 自动替换并覆盖原文件
node tools/humanize.js <小说文件.txt> --apply --inplace
```

## 禁词速查

> 完整列表见 `spec/banned-words.json`，以下为核心禁词速查。

### 情感类（必须改为具体描写）

不禁、竟然、五味杂陈、百感交集、心中一紧、不由得、不由自主、心头一震、暗自、强忍、若有所思、心领神会、恍然大悟

### 描写类（必须个性化）

目光深邃、嘴角微微上扬、微微一笑、不禁一颤、气质不凡、眉头紧锁、眼中闪过一丝、语重心长、意味深长、沉默不语

### 比喻类（必须来自角色经验）

温润如玉、静水深流、春日暖阳、刀削斧凿、深不见底、宛如、犹如、恰似、好似

### 连接类（禁止作为段落开头）

值得注意的是、与此同时、不仅如此、然而、因此、总而言之、显而易见、不言而喻、事实上、毫无疑问、众所周知、更为重要的是、需要指出的是、不可否认、诚然

## 快速自检 120 项

> 完整检测逻辑见 `tools/detect.js`，以下为检查项分类速查。

| 层级 | 项数 | 检测内容 |
|------|------|----------|
| 语言层 | 20 项 | AI 高频副词、重复句首、万能连接词、抽象名词、段落长度方差、句式单调、重复用词、成语堆砌、"的/了"字密度、万能动词/形容词、否定词密度、"是/会/能/把"字句密度、被动/排比句式、万能量词 |
| 情绪层 | 12 项 | 情绪直白词、万能表情词、情绪标签词、情绪动作支撑、环境折射、他人反应、情绪递进/反差/层次/铺垫/留白/矛盾 |
| 对白层 | 10 项 | 连续纯对白、标签多样性、潜台词、信息倾倒、长度变化、打断/犹豫、省略、动作穿插、环境穿插、冲突 |
| 描写层 | 10 项 | 五感覆盖、景物情绪关联、色彩/光影/声音/气味/温度/质感、层次、留白 |
| 节奏层 | 12 项 | 长短句交替、段落长度分布、场景切换标记、紧张短句化、详略得当、快慢交替、高潮加速、低谷停顿、过渡自然、不拖沓/仓促、节奏变化 |
| 结构层 | 12 项 | 开头钩子、章末悬念、时间线/空间标记、伏笔/呼应、悬念推进、冲突升级、转折点、高潮、结局感、余韵 |
| 角色层 | 12 项 | 视角一致性、角色数量控制、独特语言/动作、外貌/心理描写、动机/成长/矛盾/关系/弱点/选择 |
| Show vs Tell | 12 项 | 行动/细节/对话/冲突/对比/象征展示、不直接陈述情感/评价角色/解释动机/告诉读者感受、比喻词/突然不过多 |
| 深层结构层 | 20 项 | 安全比喻、明喻密度、比喻距离、信息密度方差、闲笔存在、直接讲述存在、叙述者存在感、情感节拍自然度、破折号使用、段首词多样性、句式节奏突变、细节密度适中、对白口语化、方言俗语、语气波动、场景功能混杂、时间标记自然度、因果链自然度、段落结构多样性、整体人味评估 |

### 评分标准

| 分数 | 标签 | 含义 |
|------|------|------|
| 0-15 | 人味十足 | 基本检测不出 AI 痕迹 |
| 16-30 | 轻微机器感 | 少量问题，微调即可 |
| 31-50 | 中等 AI 味 | 需要系统性修改 |
| 51-75 | 明显 AI 味 | 需要大幅重写 |
| 76-100 | 纯机器输出 | 需要重新生成 |

## 与其他 Skill 的关系

本 skill 是**第二创作者**（检测+修复），负责在写作完成后发现问题并修复。

**第一创作者**（写作 skill）负责在写作阶段就遵守规则，减少 AI 痕迹：

- `ancient-romance-skill` — 古风言情写作（含提词器、写作规则、感官词库）

写作时应先用第一创作者的规则生成初稿，再用本 skill 检测修复。
